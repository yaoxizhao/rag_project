## 一、问题分析：naive_rag 为何比 no_rag 差

对 2026-04-04 的实验结果（dev 集，n=200）进行了根因分析：

| 指标 | no_rag | naive_rag |
|------|--------|-----------|
| Answer Relevancy | 0.7790 | 0.0530 |
| Context Recall | N/A | 0.0174 |
| Context Precision | N/A | 0.0671 |
| Faithfulness | N/A | 0.2976 |

**根本原因（按优先级）：**

1. **语料库覆盖率不足（最根本）**：dev 模式仅随机采样 10k 文档，NQ ground truth 分布在全量 2.6M 篇中，命中率约 0.38%，导致 Context Recall 仅 1.74%。
2. **Prompt 强制约束**：naive_rag 模板要求模型"ONLY on provided context"，检索到无关内容时模型大量输出拒绝语句，Answer Relevancy 暴跌至 5.3%。
3. **无相似度阈值**：Retriever 无论质量如何始终返回 top_k=3，噪声段落直接注入 prompt。

**结论**：原因 1 是实验环境问题需修复；原因 2、3 是 naive_rag 的标准形态，保留作为 baseline，后续创新模块的对比变量。

## 二、代码修改

### 1. `data/loader.py` — 新增 `load_corpus_with_guaranteed_hits()`

新增函数，构建语料库时保证所有测试 query 的 relevant docs 一定包含在内，再随机填充到目标规模。

**逻辑**：
1. 调用 `load_queries_with_qrels()` 获取所有 relevant doc_ids（500 queries → 627 个 relevant docs）
2. 流式扫描 corpus.jsonl，先捞出全部 relevant docs（guaranteed=627，命中率 100%）
3. 随机采样剩余文档补足到 50,000 总量（random=49,373）

### 2. `build_index.py` — eval 模式切换新函数

eval 模式调用 `load_corpus_with_guaranteed_hits()`，dev 模式保持不变。

### 3. `config.py` — 新增 GPU 配置

新增 `CUDA_VISIBLE_DEVICES = "1"`，在 `os.environ` 中统一设置，所有脚本 `import config as cfg` 时自动生效，无需每次在命令行前缀指定。
