## 二、各文件说明

### 顶层脚本

| 文件 | 作用 |
|------|------|
| `config.py` | **全局唯一配置源**。集中定义所有超参（chunk_size、top_k、模型路径、API 密钥等），所有模块 `import config as cfg` 读取，禁止任何模块内硬编码。 |
| `serve_model.sh` | **vLLM 启动脚本**。以 TP=2 在 GPU 1&2 上启动 Qwen2.5-7B-Instruct，监听 8000 端口，提供 OpenAI 兼容接口。`--served-model-name` 固定为 `Qwen2.5-7B-Instruct`。 |
| `build_index.py` | **一次性索引构建脚本**。加载语料库（`load_corpus`）→ 切块（`chunk_corpus`）→ 写入 ChromaDB（`Retriever.add`）。支持 `--mode dev/eval` 和 `--force` 重建。 |
| `run_baseline.py` | **实验主入口**。串联 Retriever / Augmenter / Generator，支持 `--mode no_rag / naive_rag`，输出带时间戳的 CSV 文件（含 query_id、question、answer、contexts、ground_truth 五列）。 |
| `evaluate.py` | **Ragas 评估入口**。读取 `run_baseline.py` 输出的 CSV，初始化 DeepSeek LLM Judge 和本地 bge-m3 Embedding，调用 Ragas 计算指标，输出到终端和 JSON 文件。 |

### `rag/` — 核心 RAG 模块（可插拔）

| 文件 | 作用 |
|------|------|
| `rag/__init__.py` | 包初始化文件（空）。 |
| `rag/retriever.py` | **向量检索器**。封装 BAAI/bge-m3（sentence-transformers）+ ChromaDB（cosine 空间，本地持久化）。公开接口：`Retriever(collection_name)` → `.add(chunks)` / `.retrieve(query, top_k)` / `.count()` / `.reset()`。支持断点续传（部分索引时自动过滤已存在 chunk_id）。**插拔点**：后续可替换为 `HybridRetriever`（BM25+Dense）或 `RerankerRetriever`，接口不变。 |
| `rag/augmenter.py` | **Prompt 构建器**。根据 `contexts` 是否为空自动切换 no_rag / naive_rag 模板。no_rag 模板直接提问，naive_rag 模板注入编号上下文并要求仅基于上下文作答。**插拔点**：后续可替换为 `CoTAugmenter`、`HyDEAugmenter` 等，接口不变。 |
| `rag/generator.py` | **LLM 生成器**。通过 openai SDK 调用 vLLM 的 OpenAI 兼容接口，封装为 `Generator().generate(prompt) → str`。包含 `health_check()` 方法用于启动时验证服务连通性。**插拔点**：后续可替换为 DeepSeek/GPT-4o API，接口不变。 |

### `data/` — 数据处理模块

| 文件 | 作用 |
|------|------|
| `data/__init__.py` | 包初始化文件（空）。 |
| `data/loader.py` | **BeIR/NQ 数据加载器**。通过 hf-mirror.com 下载 corpus.jsonl.gz / queries.jsonl.gz / qrels/test.tsv，本地缓存在 `HF_CACHE_DIR/beir_nq/`。提供：`chunk_text()`（word-based 切块）、`load_corpus(n)`（蓄水池采样 n 篇文档）、`chunk_corpus()`（全量切块）、`load_queries_with_qrels(n)`（采样 n 条 query 并关联 ground truth）。seed=42 保证可复现。 |

### `evaluation/` — 评估模块

| 文件 | 作用 |
|------|------|
| `evaluation/__init__.py` | 包初始化文件（空）。 |

## 三、初步结果分析

naive_rag 的各项指标普遍低于预期，主要原因：

1. **检索质量差（context_recall=0.017，context_precision=0.067）**：dev 模式仅用 10k corpus，而 NQ 的 ground truth 文档分布在约 2.6M 篇全量语料中，随机采样的 10k 子集命中率极低。
2. **Answer Relevancy 下降（naive_rag=0.053 vs no_rag=0.779）**：检索到的无关上下文干扰了 Qwen2.5-7B 的回答方式，模型倾向于输出"I cannot find the answer in the provided context"这类固定语句，导致相关性得分低。
3. **Faithfulness 偏低（0.298）**：模型回答并未严格依赖检索到的上下文（因为上下文本身就不相关）。

**下一步建议**：构建 `nq_eval` 索引（50k corpus），在 eval 集（500 queries）上重新评估，结果会更真实地反映 RAG 对幻觉的影响。
